Understanding Simulated Annealing

Initial Phase

Starts with a random coloring of regions. Temperature is high (100.0), allowing the algorithm to explore freely and accept worse solutions with high probability.

Random Exploration

At each step, randomly select a region and change its color. If fewer conflicts occur, always accept. If more conflicts occur, accept with probability e-ΔE/T.

Cooling Process

Temperature decreases gradually (multiplied by 0.985 each iteration). As T approaches 0, the algorithm becomes greedier and less likely to accept worse solutions.

Convergence

Eventually converges to a near-optimal or optimal coloring. The gradual cooling helps escape local minima that a pure greedy algorithm would get stuck in.

How Simulated Annealing Works

🔥 Algorithm Insight: Randomly changes a region's color. If conflicts reduce → accept. If conflicts increase → accept with probability exp(-ΔE / T). Temperature cools down over time, reducing randomness. This avoids local minima and finds near-optimal coloring.
